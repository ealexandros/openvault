# OpenVault Brand Guidelines

## Our Philosophy

OpenVault is built on transparency and trust. While we believe in the freedom of open-source software, we must ensure that users can always identify official, audited versions of our application. These guidelines exist to prevent phishing and protect the community.

## Licensing Distinction

- **The Code:** Distributed under the [MIT License](LICENSE). You are free to copy, modify, and distribute the underlying logic.
- **The Brand:** The logo (the shield icon) is **not** covered by the MIT license.

## Guidelines for Forks and Redistribution

If you redistribute a modified version of this software (a "fork"), you **must**:

- **Remove Assets:** Remove all official OpenVault logos and icons from the UI and documentation.
- **No Impersonation:** Clearly state that your project is not the official OpenVault project and is not affiliated with the original authors.

## Contribution Exception

You are fully encouraged to fork this repository for the purpose of submitting Pull Requests. In these cases, you do not need to remove branding as long as the code is intended to be merged back into the official OpenVault project.
